domain constraint
	도메인의 값들은 모든 속성들의 경우의 수를 포함하는 집합이여야 한다.

key constraint
   한 릴레이션 내 튜플의 집합에서 중복된 슈퍼키가 존재할 수 없다.

entity integrity
    프라이머리 키 로써 사용되는 어트리뷰트는 널 상태로 존재할 수 없다.
 
referential integrity
    참조하고자 하는 포린 키의 값은 유효성을 가지며 존재하는 프라이머리 키이거나 아예 널 값 이여야 한다.



	1. 조작하고자 하는 어트리뷰트의 값이 도메인의 원소가 아닐 경우
	2. 업데이트하거나 모디파이한 키 값이 이미 존재할 경우
	3. 유효하지 안허간 무존재한 프라이머리 키를 포린 키로 사용할 경우
	4. 프라이머리 키가 널 인 경우


	1.딜리트하거나 업데이트하려는 는 튜플의 프라이머리키가 어떤 튜플에 의해 참조되는 경우





어떤 튜플을 업데이트할 때 제약조건의 위반 가능성이 있다.
대응방법은 아래와 같다.
	1. 제약조건을 위반하는 해당 연산을 취소
	2. 연산을 수행하되 유저에게 위반을 알림
	3. 캐스케이드 연산을 수행하거나 갱신되는 튜플을 참조하는 포린 키의 값을 널로 설정
	4. 유저가 정의한 에러 정정 루틴 수행
